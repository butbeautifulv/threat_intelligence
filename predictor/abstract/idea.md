Давай мыслить так, как будто у нас есть “завод” из узлов.

    Node (узел):  
    Сущность, которая что-то делает.

        принимает запросы (разных протоколов)

        может слушать очереди

        может вызывать другие узлы

        имеет конфиг и контракт (input/output)

    Port (порт):  
    Точка взаимодействия узла с внешним миром.

        входной/выходной

        протокол: http, sql, neo4j, kafka, protobuf-rpc, grpc, file, etc.

        тип сообщения: схема данных (CUE/Protobuf/JSON-schema)

    Adapter (адаптер):  
    Преобразователь протокола/формата.

        HTTP → доменная команда

        Kafka message → доменное событие

        SQL row → доменная сущность

        Neo4j result → доменная сущность

    Domain (домен):  
    Чистая логика, не знающая о протоколах.

        команды/запросы

        сущности/агрегаты

        правила

    Factory (конфиг-фабрика):  
    Узел, который не обрабатывает данные, а генерирует/собирает конфиги для других узлов.

        может собирать пайплайны

        может валидировать совместимость портов

        может генерировать артефакты (Go-код, Helm, Terraform, etc.)
